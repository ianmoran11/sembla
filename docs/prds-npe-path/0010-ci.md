# PRD 0010: Continuous integration

## Context

A project whose core promise is reproducibility should have CI that
re-verifies it on every push (`docs/sembla-assessment.md` flagged the gap).
The determinism claims are cheap to check mechanically: run twice, compare
bytes. GPU CI is out of reach (no hosted NVIDIA runners here); the GPU story
stays a documented manual path (PRD 0009's runbook) with a dispatch-only
stub.

## Goal

GitHub Actions workflows that build, lint, test, and re-verify determinism on
every push/PR, with the Lean frontend and the Python pipeline covered and the
GPU path stubbed honestly.

## Specification

- `.github/workflows/ci.yml`, jobs on `push` and `pull_request`:
  - **rust** (ubuntu-latest): toolchain pinned from `rust-toolchain.toml`,
    cargo cache, `./scripts/check.sh`, `cargo build --workspace`,
    `cargo test --workspace`.
  - **determinism** (ubuntu-latest): run the SIR example twice with the same
    seed and `diff` the results CSV, summaries CSV, and run manifest
    byte-for-byte; repeat for one `sweep` invocation. Fails on any byte
    difference.
  - **lean** (ubuntu-latest): toolchain from the frontend's pin (see
    `frontend/`), cached; build and run the frontend tests per
    `frontend/README.md`. If hosted-runner Lean setup proves impractical,
    the job may be marked `continue-on-error: false` but path-filtered to
    `frontend/**` — never silently skipped on frontend changes.
  - **npe-smoke** (ubuntu-latest, path-filtered to `calibration/**` and
    `docs/prds-npe-path/**`): install the pinned Python deps (cached), run a
    reduced-size training smoke test (tiny pair count, no SBC) plus the
    refusal tests. The full PRD-0007 acceptance config stays a local/manual
    run; the smoke test exists to catch API drift.
- **gpu-differential** workflow, `workflow_dispatch` only: a stub that
  prints the PRD-0009 remote runbook location and exits — it exists so the
  manual step has a named home, and it must fail loudly if anyone wires it to
  `push` without a runner.
- Keep total PR wall-time reasonable (caching; the determinism job reuses the
  rust job's build artifacts if straightforward, otherwise rebuilds).
- Badge in the root `README.md`.

## Non-goals

A license file (repository-owner decision, tracked outside this PRD set).
Release/publishing automation. GPU runners. Coverage/benchmark tracking.
Scheduled jobs.

## Acceptance criteria

1. Workflow YAML parses (asserted mechanically, e.g. a Python/yq parse step in
   a repo test or documented one-shot check) and `actionlint` passes if
   available locally; otherwise record *unanswered* for the lint per the
   honest-reporting convention.
2. Every CI step is runnable locally by the equivalent documented commands,
   and doing so succeeds on this machine: `./scripts/check.sh`, build, test,
   the determinism double-run diff, and the npe-smoke commands (or
   *unanswered* with reason for environment-blocked steps).
3. The determinism job's script is checked in (not inlined YAML-only), so it
   is testable locally and reusable; a local execution is demonstrated green.
4. The gpu-differential stub exists, is dispatch-only, and its body names the
   PRD-0009 runbook.
5. Root `README.md` gains the badge and one sentence on what CI checks.
